# METADATA-STANDARD

## Obligatorios

```yaml
id:
title:
type:
status:
version:
created:
updated:
owner:
source:
```

## Recomendados

```yaml
aliases:
reviewer:
language:
region:
tags:
relationships:
evidence_level:
confidence:
```

## Reglas

- Fechas en formato `YYYY-MM-DD`.
- `status` usa el ciclo oficial.
- `source` identifica documento, URL, registro o lote.
- `reviewer` es obligatorio para aprobar contenido clínico.
