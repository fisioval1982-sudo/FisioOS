# DOC-002 — Principios de Desarrollo de FisioOS

## Objetivo

Establecer los principios permanentes que regirán el desarrollo de FisioOS.

## Principios

1. **Single Source of Truth**: Airtable es la fuente operativa del conocimiento.
2. **Git es el historial oficial**.
3. **Arquitectura estable**: solo cambia mediante ADR.
4. **Conocimiento antes que funcionalidades**.
5. **Evidencia primero**.
6. **Clínica primero**.
7. **Automatización por defecto**.
8. **Revisión humana obligatoria**.
9. **Identificadores permanentes**.
10. **Simplicidad y mantenibilidad**.
