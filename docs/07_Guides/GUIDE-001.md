# Guía de estilo documental

## Objetivo

Esta guía define cómo debe escribirse toda la documentación oficial de FisioOS.

## Idioma

Toda la documentación se redactará en español.

Los nombres técnicos internacionales podrán mantenerse en inglés cuando sean el estándar.

## Formato

Todos los documentos utilizarán Front Matter.

## Identificadores

Cada documento tendrá un identificador único.

Ejemplos:

- DOC-001
- ADR-001
- SPEC-001
- GUIDE-001

## Títulos

El nombre del archivo contendrá únicamente el identificador.

El título completo irá dentro del documento.

## Versionado

Todo documento tendrá versión.

## Estado

Estados permitidos:

- Draft
- Review
- Approved
- Deprecated

## Enlaces

Siempre mediante enlaces internos de Obsidian.

Ejemplo:

[[DOC-016]]

Nunca mediante rutas del sistema operativo.
