# ADR-006 — Convención de nombres y versionado

## Estado

**Aceptado**

---

# Contexto

FisioOS utilizará cientos o miles de documentos (especificaciones, ontología, casos clínicos, entidades, investigación y documentación técnica).

Es imprescindible definir una convención estable para evitar duplicados, enlaces rotos y versiones inconsistentes.

---

# Decisión

## 1. Nombre de los archivos

Todos los documentos oficiales utilizarán un nombre permanente **sin número de versión**.

Ejemplos:

- DOC-001.md
- DOC-016.md
- SPEC-000.md
- SPEC-001.md
- ONTO-000.md
- CAT-000.md
- STR-000001.md
- PAT-000001.md
- FIND-000001.md
- TRT-000001.md

---

## 2. Versionado

La versión se almacenará únicamente en:

- El campo `version` del Front Matter.
- El historial de Git.

Ejemplo:

```yaml
version: 1.2
```

---

## 3. Archivos temporales

Durante la redacción podrán existir archivos temporales como:

- SPEC-001_v0.8.md
- CAT-000_v0.2.md

Estos archivos **nunca** se incorporarán al repositorio.

Antes de instalarse deberán renombrarse al formato oficial.

---

## 4. Commits de Git

Los mensajes de commit seguirán estas reglas:

- En inglés.
- En modo imperativo.
- Cortos y descriptivos.

Ejemplos:

- Add CAT-000 master dictionary
- Create MODULE-001 upper limb
- Update SPEC-001 metamodel
- Refine STR-000106 rotator cuff
- Fix ontology links

---

## 5. Beneficios

- Un único nombre por documento.
- Enlaces estables en Obsidian.
- Historial completo mediante Git.
- Facilita automatizaciones y exportaciones.
- Evita documentos duplicados.

---

# Consecuencias

Esta convención es obligatoria para todo el repositorio FisioOS. Cualquier excepción deberá aprobarse mediante un nuevo ADR.
