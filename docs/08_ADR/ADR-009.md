# ADR-009 — Patologías como nodos maestros de FisioOS

## Decisión

La propia entidad `PAT` actuará como nodo maestro. No se creará un HUB separado para la misma patología.

## Ubicación

`knowledge/pathology/<region>/`

## Reglas

1. Identificador permanente.
2. Sin duplicar contenido de otras entidades.
3. Enlace a publicaciones, casos, protocolos, ejercicios y servicios.
4. Las relaciones no implican indicación clínica universal.
5. Validación humana antes de `Approved`.

## Consecuencias

- Un único punto de entrada por patología.
- Menor duplicación.
- Mejor navegación, RAG, SEO, GEO y LLM.
